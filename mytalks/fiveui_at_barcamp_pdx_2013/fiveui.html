Benjamin Jones Barcamp Portland 2013

Open Source UI Tests for the Common Good

I want to talk about a tool for making UI testing easier.

User Interface Testing

  • There are many sets of UI guidelines

    1. usability.gov
    2. Apple's iOS Human Interface Guidelines,
    3. Microsoft's "UX Guide"
    4. ...

  • There are many UI testing tools

    1. Selenium
    2. WAVE (Web Accessibility),
    3. HTML Lint
    4. ...

But we're lacking something! there aren't easy ways to integrate the tools with the guidelines.

FiveUI

FiveUI is a framework and tool for checking web-based user interfaces against codified UI guidelines

FiveUI is...

  • an interactive browser extension
  • a headless testing automation framework
  • open-source (Apache 2.0)
  • at http://galoisinc.github.com/FiveUI

Goals

  • enable development of open-source UI tests
  • create an extensible framework for testing user interfaces
  • make existing and future UI guidelines machine actionable

Using FiveUI

  • anyone can encode guidelines as rules and rule sets
  • rules are literal Javascript Objects

Example:

{ 'id': 1001802
, 'name': "Don't use empty headings."
, 'description': 'Empty headings confuse layout.'
, 'rule':
  function() {
    fiveui.query(':header').each(
       function(i, elt) {
         if ($(elt).text() == '') report('Heading is empty', elt);
       }
    );
  }
}

Using FiveUI

  • once you've written a useful rule, add it to the growing collection
  • write general-purpose rules that apply to any website
  • the community can take advantage of the rules contributed by everyone else

FiveUI is useful for ...

  1. automated testing of encoded UI guidelines
  2. UI accessibility testing
